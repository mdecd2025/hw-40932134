Task5

Webots是什麼?

webots 是一套專門用來模擬行動機器人的軟體，它就像一個虛擬的機器人實驗室，可以快速幫助你設計、測試與改良機器人。你可以在裡面建構出具有真實物理特性的 3D 世界，加入各種物品與機器人，然後替機器人加裝感測器（像是攝影機、距離感測器等）和致動器（像是輪子、馬達等），再透過程式讓它們執行你想要的動作。Webots 也支援把模擬結果轉移到實體機器人上使用，例如 e-puck 或 Nao。

Webots可以做什麼?

Webots 很適合用來進行機器人領域的研究與教學。它的應用很廣，例如：

  • 設計機器人原型（無論是在學術、車用或玩具領域）

  • 研究機器人的移動方式（像是雙足步行、人形機器人等）

  • 模擬多個機器人互動（群體智慧或協同作業）

  • 開發具學習能力的機器人（如使用 AI 或演化演算法）

  • 教學、比賽訓練平台（例如 RoboCup）

使用 Webots 需要懂什麼？

要使用 Webots 來寫控制機器人的程式，建議對以下語言有基本了解：

  • C、C++、Java、Python、MATLAB

如果是新手，也可以使用 Webots 內建的圖形化程式語言 BotStudio（目前適用於部分機器人如 e-puck）。如果你想設計自己的機器人模型或場景，建議懂一些 3D 圖形概念與 VRML97 語法，這樣可以自訂外觀和結構。

要去哪裡找幫助？

Webots 提供兩種支援方式：

  • 官方支援（付費）：提供專業服務與開發協助

  • 社群支援（免費）：可在以下平台找到幫助（但不保證回應速度）：

    • Robotics StackExchange（加上 webots 標籤）

    • GitHub Discussions（問技術問題）

    • GitHub Issues（回報錯誤）

    • Discord（快速聊天討論）

Webots 模擬是怎麼組成的？

一個 Webots 模擬專案會包含：

  • 一個 世界檔（.wbt）：描述場景、物件與機器人

  • 一或多個 控制程式：讓機器人執行任務的程式碼

  • 可選的 物理插件：進一步調整物理模擬行為

  • 外部資源檔案（例如模型、貼圖等）

什麼是「世界」？

在 Webots 裡，「世界」就是一個 3D 場景設定，定義了機器人和所有物件的樣貌、位置和物理特性。這些設定儲存在 .wbt 檔案中，是一種層級式的描述方式（結構有點像 VRML97）。這個檔案會指定每個機器人要用哪個控制程式，但不包含實際的程式內容。

什麼是「控制器」？

控制器就是你寫給機器人的「大腦」，用來控制它怎麼移動、感測、決策。每個控制器是獨立的程式，Webots 會在模擬一開始時自動為每個機器人開一個控制器進程。控制器可以用 C/C++、Java、Python 或 MATLAB 撰寫，放在專案資料夾裡的 controllers 目錄中。

什麼是「監督控制器」？

如果你想讓某個控制器有更高的權限，例如操控整個模擬、直接移動機器人或錄影，那就可以設定它為「監督控制器（Supervisor）」。只要在機器人設定中把 supervisor 欄位設成 TRUE，就可以讓它進行這些高階操作。監督控制器同樣也支援所有語言撰寫。
