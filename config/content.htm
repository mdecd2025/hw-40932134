假設您希望在 Webots 中建立一個簡單場景：一個機器人在平面上移動並避開障礙物。以下是手把手說明如何透過圖形介面完成：
步驟 1：啟動 Webots
  1. 打開 Webots 應用程式。
  2. 在啟動畫面選擇「New World」（新建世界）。
步驟 2：設置基本場景
  1. 加入地板
    • 在頂部選單點擊「Add」（添加）。
    • 選擇「Solid」（實體物件） > 「Box」（方塊）。
    • 在屬性面板中設置：
      • 大小（Size）：10m x 10m x 0.1m（寬廣的平面）。
      • 名稱（Name）：Floor。
    • 注意：確保地板位於 Z=0，避免懸浮。
  2. 加入機器人
    • 點擊「Add」 > 「Robot」（機器人）。
    • 在下拉選單中選擇「E-puck」（一個簡單的雙輪機器人）。
    • 調整位置（Translation）：例如 (0, 0, 0.05)，使其位於地板上。
    • 注意：檢查機器人是否與地板接觸，避免穿透或漂浮。
步驟 3：添加控制器
  1. 點擊機器人節點（E-puck）。
  2. 在屬性面板中找到「Controller」欄位。
  3. 點擊下拉選單，選擇「<extern>」（外部控制器）。
  4. 點擊「Wizard」（精靈），選擇「Create a new controller」。
  5. 輸入名稱（如「avoid_obstacle」），選擇語言（如 Python）。
  6. Webots 會生成一個簡單的控制器檔案。

步驟 4：編輯控器

       1.保存檔案。
步驟 5：運行模擬
  1. 點擊頂部選單的「Run」（運行按鈕）。
  2. 觀察機器人是否能移動並避開障礙物。
  3. 若有問題，檢查：
    • 機器人與地板的接觸是否正常。
    • 感測器是否啟用。
    • 控制器是否正確綁定。
注意事項
  • 坐標系統：Webots 使用 X（左右）、Y（前後）、Z（上下）的坐標系，確保物件位置合理。
  • 物理屬性：檢查物件的質量與摩擦係數，避免異常滑動。
  • 控制器同步：確保控制器與機器人節點正確關聯。

複習webots SolidReference 節點的用法說明，用在 Joint 節點的 endPoint 欄位中，用來指向一個現有的 Solid 或靜態環境 <static environment>。通過這種方式，可以閉合機械迴圈。

當指向一個 Solid 時，唯一的限制是 Solid 和 Joint 必須是某個共同上層 Solid 的下層。

意思即是指 SolidReference 是一個引用類型，允許在 Joint 節點的 endPoint 欄位中指定某個目標。

它可以指向兩種類型的對象：

  1. 現有的 Solid：這是一個具體的實體，但這個 Solid 和 Joint 必須是某個共同上層 Solid 的下層。
  2. 靜態環境 <static environment>：這通常指的是不動的參考框架（例如地面或固定基座）

第一作法：

若將 fourbar linkage 的 joint4 設置在 base link 的最右端，然後將 joint4 的 endPoint 欄位，設為 SolidReference 指向 link3，即可符合 Soild (link3) 與 Joint (joint4) 都是某個共同上層 Solid (base) 的下層。

第二作法：

可以將 fourbar linkage 的 joint4 設置在 link3 的末端，然後將 joint4 的 endPoint 欄位指向 <static environment>，讓 link3 的末端固定在 ground (其作用與固定在 base 節點相同)。

Task5

Webots是什麼?

webots 是一套專門用來模擬行動機器人的軟體，它就像一個虛擬的機器人實驗室，可以快速幫助你設計、測試與改良機器人。你可以在裡面建構出具有真實物理特性的 3D 世界，加入各種物品與機器人，然後替機器人加裝感測器（像是攝影機、距離感測器等）和致動器（像是輪子、馬達等），再透過程式讓它們執行你想要的動作。Webots 也支援把模擬結果轉移到實體機器人上使用，例如 e-puck 或 Nao。

Webots可以做什麼?

Webots 很適合用來進行機器人領域的研究與教學。它的應用很廣，例如：

  • 設計機器人原型（無論是在學術、車用或玩具領域）

  • 研究機器人的移動方式（像是雙足步行、人形機器人等）

  • 模擬多個機器人互動（群體智慧或協同作業）

  • 開發具學習能力的機器人（如使用 AI 或演化演算法）

  • 教學、比賽訓練平台（例如 RoboCup）

使用 Webots 需要懂什麼？

要使用 Webots 來寫控制機器人的程式，建議對以下語言有基本了解：

  • C、C++、Java、Python、MATLAB

如果是新手，也可以使用 Webots 內建的圖形化程式語言 BotStudio（目前適用於部分機器人如 e-puck）。如果你想設計自己的機器人模型或場景，建議懂一些 3D 圖形概念與 VRML97 語法，這樣可以自訂外觀和結構。

要去哪裡找幫助？

Webots 提供兩種支援方式：

  • 官方支援（付費）：提供專業服務與開發協助

  • 社群支援（免費）：可在以下平台找到幫助（但不保證回應速度）：

    • Robotics StackExchange（加上 webots 標籤）

    • GitHub Discussions（問技術問題）

    • GitHub Issues（回報錯誤）

    • Discord（快速聊天討論）

Webots 模擬是怎麼組成的？

一個 Webots 模擬專案會包含：

  • 一個 世界檔（.wbt）：描述場景、物件與機器人

  • 一或多個 控制程式：讓機器人執行任務的程式碼

  • 可選的 物理插件：進一步調整物理模擬行為

  • 外部資源檔案（例如模型、貼圖等）

什麼是「世界」？

在 Webots 裡，「世界」就是一個 3D 場景設定，定義了機器人和所有物件的樣貌、位置和物理特性。這些設定儲存在 .wbt 檔案中，是一種層級式的描述方式（結構有點像 VRML97）。這個檔案會指定每個機器人要用哪個控制程式，但不包含實際的程式內容。

什麼是「控制器」？

控制器就是你寫給機器人的「大腦」，用來控制它怎麼移動、感測、決策。每個控制器是獨立的程式，Webots 會在模擬一開始時自動為每個機器人開一個控制器進程。控制器可以用 C/C++、Java、Python 或 MATLAB 撰寫，放在專案資料夾裡的 controllers 目錄中。

什麼是「監督控制器」？

如果你想讓某個控制器有更高的權限，例如操控整個模擬、直接移動機器人或錄影，那就可以設定它為「監督控制器（Supervisor）」。只要在機器人設定中把 supervisor 欄位設成 TRUE，就可以讓它進行這些高階操作。監督控制器同樣也支援所有語言撰寫。
