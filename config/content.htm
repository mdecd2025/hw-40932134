w5

假設您希望在 Webots 中建立一個簡單場景：一個機器人在平面上移動並避開障礙物。以下是手把手說明如何透過圖形介面完成：
步驟 1：啟動 Webots
  1. 打開 Webots 應用程式。
  2. 在啟動畫面選擇「New World」（新建世界）。
步驟 2：設置基本場景
  1. 加入地板
    • 在頂部選單點擊「Add」（添加）。
    • 選擇「Solid」（實體物件） > 「Box」（方塊）。
    • 在屬性面板中設置：
      • 大小（Size）：10m x 10m x 0.1m（寬廣的平面）。
      • 名稱（Name）：Floor。
    • 注意：確保地板位於 Z=0，避免懸浮。
  2. 加入機器人
    • 點擊「Add」 > 「Robot」（機器人）。
    • 在下拉選單中選擇「E-puck」（一個簡單的雙輪機器人）。
    • 調整位置（Translation）：例如 (0, 0, 0.05)，使其位於地板上。
    • 注意：檢查機器人是否與地板接觸，避免穿透或漂浮。
步驟 3：添加控制器
  1. 點擊機器人節點（E-puck）。
  2. 在屬性面板中找到「Controller」欄位。
  3. 點擊下拉選單，選擇「<extern>」（外部控制器）。
  4. 點擊「Wizard」（精靈），選擇「Create a new controller」。
  5. 輸入名稱（如「avoid_obstacle」），選擇語言（如 Python）。
  6. Webots 會生成一個簡單的控制器檔案。

步驟 4：編輯控器

       1.保存檔案。
步驟 5：運行模擬
  1. 點擊頂部選單的「Run」（運行按鈕）。
  2. 觀察機器人是否能移動並避開障礙物。
  3. 若有問題，檢查：
    • 機器人與地板的接觸是否正常。
    • 感測器是否啟用。
    • 控制器是否正確綁定。
注意事項
  • 坐標系統：Webots 使用 X（左右）、Y（前後）、Z（上下）的坐標系，確保物件位置合理。
  • 物理屬性：檢查物件的質量與摩擦係數，避免異常滑動。
  • 控制器同步：確保控制器與機器人節點正確關聯。

複習webots SolidReference 節點的用法說明，用在 Joint 節點的 endPoint 欄位中，用來指向一個現有的 Solid 或靜態環境 <static environment>。通過這種方式，可以閉合機械迴圈。

當指向一個 Solid 時，唯一的限制是 Solid 和 Joint 必須是某個共同上層 Solid 的下層。

意思即是指 SolidReference 是一個引用類型，允許在 Joint 節點的 endPoint 欄位中指定某個目標。

它可以指向兩種類型的對象：

  1. 現有的 Solid：這是一個具體的實體，但這個 Solid 和 Joint 必須是某個共同上層 Solid 的下層。
  2. 靜態環境 <static environment>：這通常指的是不動的參考框架（例如地面或固定基座）

第一作法：

若將 fourbar linkage 的 joint4 設置在 base link 的最右端，然後將 joint4 的 endPoint 欄位，設為 SolidReference 指向 link3，即可符合 Soild (link3) 與 Joint (joint4) 都是某個共同上層 Solid (base) 的下層。

第二作法：

可以將 fourbar linkage 的 joint4 設置在 link3 的末端，然後將 joint4 的 endPoint 欄位指向 <static environment>，讓 link3 的末端固定在 ground (其作用與固定在 base 節點相同)。

各週進度:

第一週到第三週
這段期間主要是建立可攜式開發環境，並透過 Python 撰寫程式來協助課程分組名單的分類作業。透過這個過程，不僅熟悉了基本的程式操作，也加強了對資料處理的理解。

第四週到第五週
進一步學習如何設定自己的 IPv6 網址，並利用 .bat 批次檔來進行快速設定，讓整體流程更有效率。雖然這期間也有接觸到 Webots，但大多是概念上的認識，還沒有實際操作。

第六週到第九週（目前進行中）
課程的重點轉移到 Webots 模擬平台的應用，專注於連桿設計與動作模擬，開始進行實際操作與測試。同時也輔助學習 SolveSpace 和 Blender，作為設計模型與結構建置的工具，提升整體模擬的表現與精度。

期中心得:

經過這九週的課程學習，我對整體的系統環境建置與模擬操作有了更清楚的脈絡。從一開始透過 Python 程式處理分組名單，到中期進行 IPv6 的設定，讓我逐步理解網路與程式的結合應用，並學會使用批次檔提升操作效率。

進入後期的 Webots 模擬操作，更讓我體會到模擬環境在機器人開發上的重要性。不只是單純操作模擬介面，還需要搭配幾何建模工具如 SolveSpace 和 Blender，讓整體機構更貼近實際設計。雖然這些工具在剛接觸時稍顯複雜，但隨著練習次數增加，我也慢慢掌握了其中邏輯與使用方式。

整體來說，這段學習歷程讓我從程式邏輯、網路設定到模擬操作都有所接觸，不僅提升了實作能力，也讓我對未來智慧機電整合應用有了更具體的想像與期待。
